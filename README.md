# Раздолье — поздравление Андрею Грибкову

Мини-игра-поздравление для запуска как Telegram Mini App. Пользователь нажимает
на логотип компании «Раздолье», и при каждом нажатии появляется новая
поздравительная фраза. После десятой фразы запускается праздничный финал
с конфетти и подписью «Команда Раздолья поздравляет!».

Проект полностью автономен: чистые **HTML5 / CSS3 / JavaScript**, без
сборщиков, фреймворков и серверной части.

## Структура проекта

```
razdolie/
├── index.html          — разметка страницы
├── styles.css           — стили и анимации
├── script.js             — логика игры
├── assets/
│   └── razdolie-logo.png — логотип компании
└── README.md
```

## 1. Как запустить проект локально (быстрый способ)

Проще всего — открыть `index.html` двойным щелчком в браузере. Для базовой
проверки фраз, анимаций и конфетти этого достаточно.

## 2. Почему обычное открытие index.html может иметь ограничения

При открытии файла напрямую (протокол `file://`) некоторые браузеры
ограничивают:

- загрузку изображений и скриптов из-за политики безопасности (CORS/`file://`);
- работу некоторых API (например, часть функций Telegram SDK рассчитана на
  `https://`-адрес и просто не будет активна без него);
- корректную имитацию реального окружения Telegram, которое всегда открывает
  Mini App по HTTPS-ссылке.

Поэтому для полноценного тестирования (и обязательно для запуска в Telegram)
нужен локальный сервер или HTTPS-хостинг.

## 3. Как запустить через простой локальный сервер

Выберите любой удобный вариант, находясь в папке проекта:

**Python 3 (есть почти везде):**
```bash
cd razdolie
python3 -m http.server 8000
```
Откройте `http://localhost:8000` в браузере.

**Node.js (пакет serve):**
```bash
cd razdolie
npx serve .
```

**VS Code:** расширение *Live Server* → «Go Live».

## 4. Как разместить проект на GitHub Pages

1. Создайте новый репозиторий на GitHub и загрузите в него все файлы проекта
   (`index.html`, `styles.css`, `script.js`, папку `assets/`, `README.md`).
2. В репозитории откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите источник **Deploy from a
   branch**, ветку `main` и папку `/ (root)`.
4. Сохраните настройки — через 1–2 минуты сайт будет доступен по адресу вида
   `https://<ваш-логин>.github.io/<название-репозитория>/`.
5. Этот HTTPS-адрес и есть ссылка на вашу мини-игру.

## 5. Как разместить на Netlify или Vercel

**Netlify:**
1. Зарегистрируйтесь на [netlify.com](https://netlify.com).
2. На главной странице выберите **Add new site → Deploy manually** и
   перетащите папку проекта, либо подключите GitHub-репозиторий через
   **Import from Git**.
3. Netlify автоматически развернёт статический сайт и выдаст HTTPS-адрес
   вида `https://<имя-проекта>.netlify.app`.

**Vercel:**
1. Зарегистрируйтесь на [vercel.com](https://vercel.com).
2. Нажмите **Add New → Project** и подключите GitHub-репозиторий (или
   загрузите папку через Vercel CLI: `npx vercel`).
3. Так как проект статический, дополнительная настройка сборки не нужна —
   просто подтвердите деплой. Вы получите адрес вида
   `https://<имя-проекта>.vercel.app`.

## 6. Как создать Telegram-бота через BotFather

1. Откройте в Telegram чат **@BotFather**.
2. Отправьте команду `/newbot`.
3. Укажите имя бота (отображаемое) и username (должен заканчиваться на `bot`,
   например `razdolie_congrats_bot`).
4. BotFather пришлёт токен доступа — сохраните его в надёжном месте (для
   этого проекта он не нужен, так как игра не обращается к серверу, но
   пригодится для дальнейшей настройки бота).

## 7. Как связать HTTPS-адрес мини-игры с кнопкой меню Telegram-бота

1. В чате с **@BotFather** отправьте `/mybots` и выберите вашего бота.
2. Перейдите в **Bot Settings → Menu Button** (или **Настройки → Кнопка
   меню**).
3. Выберите **Edit Menu Button URL** (или аналогичный пункт) и вставьте
   HTTPS-адрес, полученный на шаге размещения (GitHub Pages / Netlify /
   Vercel), например:
   `https://username.github.io/razdolie/`.
4. Задайте текст кнопки, например «🎉 Поздравить Андрея».
5. Откройте чат с ботом в Telegram — рядом со строкой ввода появится кнопка
   меню, которая открывает мини-игру внутри Telegram.

Также можно создать отдельную inline-кнопку с `web_app`-ссылкой через
`@BotFather → /newapp`, если нужен именно формат Telegram Mini App с
отдельным названием и иконкой.

## 8. Как заменить логотип

1. Подготовьте новое изображение в формате PNG (желательно с прозрачным
   фоном) и назовите его `razdolie-logo.png`, либо измените имя файла в
   `index.html`.
2. Замените файл в папке `assets/`.
3. Если имя файла отличается, обновите путь в `index.html`:
   ```html
   <img id="logoBtn" class="logo" src="assets/ваш-файл.png" ... />
   ```
4. Пропорции и размер логотипа адаптируются автоматически — дополнительно
   ничего пересчитывать не нужно.

## 9. Как изменить список фраз

Откройте `script.js` и найдите константу `PHRASES` в самом начале файла:

```js
var PHRASES = [
  "Андрей Грибков",
  "Основатель и вдохновитель",
  // ...
  "С днём рождения, Андрей Александрович Грибков!"
];
```

- Отредактируйте текст любой строки или добавьте/удалите строки — счётчик
  «X из Y» и прогресс-бар пересчитаются автоматически, так как `TOTAL`
  берётся из длины массива.
- Порядок фраз в массиве — это порядок их появления в игре.

## 10. Как изменить фирменные цвета

Откройте `styles.css` и отредактируйте переменные в блоке `:root` в самом
начале файла:

```css
:root {
  --navy-deep: #040c1f;   /* фон приложения */
  --navy-base: #0a1f4d;   /* основной синий логотипа */
  --blue-bright: #2f6fe0; /* акцент, свечение, круги от нажатия */
  --blue-pale: #bcd6ff;   /* светлый текст подсказок */
  --gold: #d8b169;        /* золотой акцент финала */
  --gold-soft: #f0d9a6;   /* золотой текст финальной подписи */
}
```

Изменение переменных автоматически применится ко всем элементам интерфейса.

## 11. Как проверить игру вне Telegram

1. Запустите локальный сервер (см. пункт 3) или откройте опубликованную
   ссылку (GitHub Pages / Netlify / Vercel) в обычном браузере — на
   компьютере, телефоне или планшете.
2. Проверьте:
   - последовательное появление всех 10 фраз при клике/тапе по логотипу;
   - работу с клавиатуры (Tab → фокус на логотипе, затем Enter или Space);
   - запуск финальной сцены и конфетти после десятой фразы;
   - кнопку «Посмотреть ещё раз» — она должна сбрасывать игру без перезагрузки
     страницы.
3. Так как код обёрнут в проверки `if (tg) { ... }`, при отсутствии Telegram
   SDK ошибок в консоли быть не должно — игра работает как обычный сайт.

## 12. Как проверить игру внутри Telegram

1. Опубликуйте проект на HTTPS-хостинге (GitHub Pages / Netlify / Vercel).
2. Привяжите ссылку к кнопке меню бота, как описано в пункте 7.
3. Откройте чат с вашим ботом в приложении Telegram (на iPhone, Android или
   в десктопном клиенте) и нажмите кнопку меню — мини-игра откроется внутри
   Telegram на весь экран (`Telegram.WebApp.expand()`).
4. Проверьте:
   - что логотип и текст корректно помещаются на экране без горизонтальной
     прокрутки;
   - что учитываются безопасные отступы на iPhone с вырезом (Dynamic
     Island/notch);
   - что при нажатии на логотип ощущается лёгкая вибрация (если устройство
     и версия Telegram её поддерживают);
   - что кнопка «Посмотреть ещё раз» и вся мини-игра работают так же, как и
     в обычном браузере.

## Технические детали

- Логотип не перерисовывается и не искажается — используется исходный файл
  `assets/razdolie-logo.png` с сохранением пропорций (`height: auto`).
- Конфетти реализовано собственным лёгким кодом на `<canvas>` без внешних
  библиотек — ограниченное число частиц и автоматическая остановка анимации,
  чтобы не перегружать мобильные устройства.
- Поддерживается `prefers-reduced-motion`: при включённой настройке в
  системе анимации отключаются или сводятся к минимуму.
- Повторное нажатие во время анимации смены фразы блокируется примерно на
  550 мс, а также используется защита от «двойного» события на мобильных
  устройствах (`pointerup` + отмена `touchend`).
- Игра не отправляет никаких данных на сервер и не запрашивает персональные
  данные пользователя.
